# 日志与网络页面窄屏响应式修复

## Summary

当前桌面前端的日志页与网络页在内容区宽度收窄后出现两类明显响应式问题：

- trace 表格列宽分配失衡，`TOKENS` 与 `费用 / 状态` 之间出现异常空白，最右列被挤出可视区
- 页面进入窄屏断点后，侧边栏转顶部导航与主内容区布局失衡，主卡片没有占满宽度，导致右侧出现大片无效留白；同时筛选区和列表内容被过度压缩

本任务只做最小前端 CSS 修复，不改后端数据结构、接口协议或页面信息架构。

## Requirements

- 修复日志页表格列宽分配，移除不合理的空白占比，保证最后一列完整可见
- 修复网络页表格在窄宽度下的列宽与可见性问题
- 在窄屏下让日志页与网络页主内容卡片占满可用宽度，不再出现右侧大面积留白
- 侧边栏切换为顶部导航后保持整齐对齐，避免按钮凌乱堆叠
- 筛选 chip 区域允许自然换行，但不应挤压表格主体到不可读
- trace 列表容器在窄屏下必须保留 `overflow-x: auto`，通过横向滚动查看完整列内容
- 每行 trace 数据需要有稳定的最小宽度，不能无限压缩到文字重叠
- 保持现有 i18n 文案键与组件结构不变，优先用现有 class 扩展修复

## Non-Goals

1. 不重做日志页/网络页视觉风格。
2. 不新增前端路由或状态层。
3. 不调整 trace 详情弹层交互。
4. 不修改日志/网络数据查询逻辑。

## Acceptance Criteria

1. 日志页在收窄内容区后，`费用 / 状态` 列仍可通过容器内横向滚动完整查看。
2. 网络页在收窄内容区后，末列不会被异常空白挤出。
3. `TOKENS` 与右侧状态列之间不再存在明显异常留白。
4. 在窄屏断点下，主内容区卡片宽度占满容器，不再被压在左侧。
5. 顶部导航在窄屏下保持规整换行/网格布局，无明显错位。
6. 筛选按钮区域在窄屏下可换行，但不会导致主列表内容坍塌。
7. 表格行内容在窄屏下不会被压成重叠文本；完整信息通过横向滚动访问。
